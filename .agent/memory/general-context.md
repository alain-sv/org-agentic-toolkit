# General Context

This document contains shared context applicable to ALL teams in the organization.

## Organization Overview

We are a technology organization focused on building scalable, maintainable software solutions.

## Common Terminology

- **ADR**: Architecture Decision Record
- **CI/CD**: Continuous Integration/Continuous Deployment
- **LGTM**: Looks Good To Me (approval)
- **WIP**: Work In Progress
- **MVP**: Minimum Viable Product

## Communication Channels

- **Slack**: Primary communication channel
- **GitHub**: Code repository and issue tracking
- **Jira**: Project management and ticket tracking

## Development Workflow

1. Create feature branch from develop
2. Implement feature with tests
3. Create pull request
4. Address review feedback
5. Merge after approval
6. Deploy to staging
7. Verify in staging
8. Deploy to production

## Code Review Guidelines

- Be constructive and respectful
- Focus on code quality, not personal preferences
- Explain the "why" behind suggestions
- Approve when code meets standards
- Request changes when standards are not met

## Onboarding

New team members should:
1. Read the constitution
2. Review team-specific context
3. Set up development environment
4. Complete onboarding checklist
5. Pair with experienced team member
